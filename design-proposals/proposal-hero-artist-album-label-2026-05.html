Structure ACF — Champs requis

Champ ACF Type Statut Affichage hero Notes
artisteNouveau text Recommandé Ligne au-dessus du H1 · classe an-hero__artist Remplace la partie artiste du titre WP. Texte seul, pas de markup.
— Titre WP — Post title Modifié éditoriale­mentÉditorial <h1> · classe an-hero__title Ne contient plus que le nom de l'album. Ex : "Sundowner" (sans "Kevin Morby |").
labelsExistant text Renommé en back-officeLabel UI Variante A : ligne an-hero__record · Variante B : sur même ligne que artiste · Variante C : pill distincte La clé ACF labels reste inchangée — seul le label UI dans le groupe ACF change (→ "Label discographique").
Migration éditoriale
Les chroniques existantes ont le titre WP au format "Artiste | Album". Deux approches :
  1. Remplir manuellement le champ artiste sur chaque chronique (safe, pas de modification titre).
  2. Nettoyer les titres WP (enlever la partie artiste) + remplir artiste en même temps. Plus propre pour le SEO mais requiert une passe éditoriale.
⚠ Le champ artiste est conditionnel — si vide, rien ne s'affiche (pas de bloc vide). Les anciens posts sans ce champ restent affichables.
