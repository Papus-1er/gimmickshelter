Option A Header unifié horizontal — Logo gauche · Nav centre · Recherche droite
Principe : Une seule bande, fond bleu GS (#000B29), sticky immédiatement. Logo à gauche, nav centrée, champ recherche à droite.

✔ Avantages : Pattern webzine/magazine standard · Sticky dès le départ (nav toujours accessible) · Recherche intégrée · Code simplifié (un seul nav, responsive via CSS)
✘ Compromis : Logo plus petit qu'en plein écran · Perd l'impact visuel du grand logo rouge actuel

Option B Header magazine deux niveaux — Bandeau rouge logo + barre nav noire sticky
Principe : Conserve l'identité rouge du bandeau logo actuel mais le distingue clairement de la nav (fond noir). La nav seule devient sticky. Ajout : réseaux sociaux à droite du logo, recherche à gauche.

✔ Avantages : Évolution douce de l'existant · Hiérarchie visuelle claire logo / nav · Titres en Anton dans la nav (plus "rock") · Réseaux sociaux visibles
✘ Compromis : Plus haut que l'option A · Le bandeau rouge disparaît au scroll (seule la nav noire reste visible)

Option C Header transparent sur hero → compact bleu au scroll
Principe : Sur les pages avec image/vidéo hero (front-page, archives), le header est transparent et se fond dans le visuel. Au scroll, il devient une barre compacte bleu GS sticky. Logo centré grand → logo petit à gauche.

✔ Avantages : Impact visuel fort en arrivant · Très courant sur les sites musicaux · Gain d'espace vertical · Elegant sur la homepage
✘ Compromis : Complexité JS (détection scroll) · Ne fonctionne bien que sur les pages avec hero image · Les pages sans hero doivent fallback sur la version scrollée
